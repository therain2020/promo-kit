# 配图设计规范

social-media-cn-image 模板的设计规范。所有模板遵循此文档的配色、字体、排版约定。

---

## 设计原则

1. **移动优先** — 所有模板以 750px 宽度设计，适应手机竖版浏览
2. **一图一意** — 每张图只传达 1-3 个信息点，不堆砌
3. **品牌一致** — 同一项目的所有配图保持配色和风格统一
4. **可读性第一** — 手机屏幕上文字清晰可读，对比度足够
5. **不依赖外部资源** — 所有 CSS 内联，字体用 system-ui 栈

## 排版规范

### 安全区域

- 上下各留 40px margin
- 左右各留 40px padding
- 重要内容（标题、CTA）放在上半部分（top 10%-60%）
- 底部 20% 留给次要信息或装饰

### 间距

| 元素间距 | 值 |
|---------|-----|
| 标题与正文 | 24-32px |
| 段落之间 | 20-24px |
| 列表项之间 | 16-20px |
| 卡片之间 | 20-24px |
| 图标与文字 | 12-16px |

### 卡片样式

```css
.card {
  background: #F8F8F8;
  border-radius: 16px;
  padding: 28px 24px;
  margin-bottom: 20px;
}
```

## 模板设计要点

### cover-xhs（小红书封面）

- 大号标题居中，占画面 40%-50%
- 项目名称在标题下方，28px
- 底部放一句话价值主张
- 背景可用渐变色或纯色 + 装饰元素
- emoji 可作为视觉锚点

### comparison（对比图）

- 左右或上下对比布局
- 左侧/上方: "其他方案"，右侧/下方: "本方案"
- 用 ✅ 和 ❌ 标记差异
- 用绿色和红色区分优劣
- 表格或卡片式排列

### terminal（终端截图）

- 模拟终端窗口：顶部标题栏（红黄绿三个圆点）
- 深色背景 `#0D1117`
- `$ ` 前缀的命令行
- 语法高亮的关键输出
- 左下角可选光标闪烁效果（截图时不重要）

### steps（步骤图）

- 纵向排列，每步一个卡片
- 步骤编号用大号彩色数字（主色）
- 每步：标题 + 一句话说明
- 步骤间用竖线或箭头连接

### cta-card（CTA 卡片）

- 大号主色按钮，居中
- 按钮文字：⭐ Star on GitHub / 试试看 / 下载使用
- 按钮上方：一句话价值主张
- 按钮下方：GitHub URL 或其他链接
- 可选：二维码占位区域

### phone-text（手机文案）

- 模拟手机通知/消息样式
- 圆角卡片，浅灰背景
- 可包含 emoji、短句、链接
- 适合展示"发布后的效果预览"

### architecture（架构图）

- 简单的流程图/架构图
- 用方框 + 箭头表示组件关系
- 配色克制，主色 + 灰色系
- 每个组件标注名称和一句话说明

### code-block（代码块）

- 深色背景代码区域
- 顶部文件名标签（如 `config.yaml`）
- 语法高亮：关键字、字符串、注释用不同颜色
- 可选择性展示行号
- 关键行可用高亮背景标注

## 生成参数

| 参数 | 小红书 | 小黑盒 |
|------|--------|--------|
| Viewport | 750×1000 | 750×1000 或 1200×800 |
| Device Scale | 2x (Retina) | 2x |
| 输出格式 | PNG | PNG |
| 背景色 | 白色/浅色系 | 白色或深色系 |
